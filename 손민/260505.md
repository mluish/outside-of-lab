# GStreamer

> 영상/음성 데이터를 받고, 처리하여, 내보내는 파이프라인을 생성하는 라이브러리
카메라 입력 → 인코딩 → 네트워크 송출
Source → Filter → Sink
> 

### 핵심 구성 요소

- **Element**
    
    > Element 간의 연결로 pipe-line 구성
    > 
    > - Source → 카메라, 파일, 마이크 등 **데이터 생성 주체**
    > - Filter → 인코딩, 디코딩, 크기 변환 등 **데이터 변환/처리**
    > - Sink → 화면, 파일, 네트워크 등 **데이터 최종 출력 주체**
    
- **Pad**
    
    > Element 간의 연결 접점 (Interface)
    src pad (Input) ↔ sink pad (Output) 형태의 연결
    > 
    
- **Pipe-line**
    
    > Element를 담는 컨테이너
    > 
    > 
    > **Pipeline
    > └── Source
    > └── Filter
    > └── Sink**
    > 위 와 같은 구조, 각 Element 전체를 관리함
    > 실행, 정지, 에러 처리 등
    > 

- **Bus**
    
    > Pipe-line 내부에서 일어나는 일을 내 코드에게 알려주는 메시지 시스템
    ex) pipeline → **(error occur**) → Bus → 내 코드에 알림
    >

## Pipe-line example

```jsx
gst-launch-1.0 v412src ! x264enc ! rtph264pay ! udpsink
# v412src -> 카메라에서 영상 읽기
# x264enc -> H.264로 인코딩
# rtph264pay -> RTP 패킷화
# udpsink -> 네트워크로 송출 

! 로 각 단계를 연결
```

## GStreamer 동작 순서

```jsx
1. 초기화
	gst.init()
	-> GStreamer Start
	
2. element 생성
	gst_element_factory_make()
	-> 필수 요소 생성 source, filter, sink

3. pipe-line에 생성된 element 추가
	gst_bin_add_many()

4. element connect
	gst_element_link_many()
	-> element 간 연결

5. 실행 
	gst_element_set_state(GST_STATE_PLAYING)
	-> pipe-line 가동

6. 대기
	gst_bus_timed_pop_filtered()
	-> 에러나 종료 메시지 기다림 -> GStreamer 동작 중인 상태

7. 정리
	gst_element_set_state(GST_STATE_NULL)
	gst_object_unref()
	-> 메모리 정리
```

## 상태 개념

→ GStreamer pipe-line state

- **NULL →** 초기 상태
- **READY →** 리소스 준비됨 (카메라 on 등)
- **PAUSED →** 데이터 스트림 준비됨, 일시정지
- **PLAYING →** 실제로 스트리밍 중인 상태\

**NULL → READY → PAUSED → PLAYING (실행 중인 상태 흐름)**

**PLAYING → PAUSED → READY → NULL (종료)**

#### 소켓 개념과의 차이

**소켓** 

- 데이터를 네트워크로 주고 받는 인터페이스
- 데이터 종류 무관
- 영상 처리 기능 X

**GStreamer**

- 영상/음성 특화 pipe-line
- 인코딩, 디코딩, 포맷 변환 등 내장
- 내부적으로는 소켓을 활용 → 네트워크 전송

💡 **GStreamer** = 복잡한 영상 처리의 **추상화** 가능 → only pipe-line 연결
