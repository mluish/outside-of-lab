# ☑️ 왜 Android 메인 스레드는 하나인가

> "Android에서 메인 스레드를 하나만 두는 이유는 크게 세 가지입니다.
>
> - **첫째, UI 툴킷이 thread-unsafe하게 설계되어 있어서 View 조작마다 lock을 걸면 동기화 비용으로 프레임 성능이 무너지기 때문.**
> - **둘째, Choreographer가 VSYNC에 맞춰 input → animation → measure → layout → draw를 한 스레드에서 직렬 실행해야 프레임 일관성이 보장되기 때문.**
> - **셋째, Looper와 MessageQueue 구조로 모든 이벤트가 큐에 줄 서서 순차 처리되어야 race condition 없이 실행 순서를 예측할 수 있기 때문**
>
> 정리하면, 메인 스레드가 하나인 것은 성능을 포기한 결정이 아니라, 오히려 lock 비용을 없애고 프레임과 이벤트의 일관성을 유지하기 위한 설계

-----
- thread-safe :여러 스레드가 동시에 같은 객체·자원에 접근해도 race condition이나 데이터 손상 없이 올바르게 동작하도록 설계된 상태
- VSYNC: 디스플레이가 한 프레임 그리기를 끝내고 다음 프레임을 받을 준비가 됐다고 알리는 하드웨어 신호 (60Hz면 약 16.6ms마다, 120Hz면 약 8.3ms마다 발생).
