## Android 화면 회전 설정 정리

### 1. Manfiest : 정적 선언
- Activity 최초 방향을 선언하는 곳
 
```
<activity
    android:name=".MainActivity"
    android:screenOrientation="portrait" />
```

### 2. ActivityInfo: 메타정보 클래스
- Manifest의 <activity> 태그 내용을 코드에서 읽을 수 있게 담아둔 클래스

```kotlin
// 현재 Activity의 회전 설정값 읽기
val info = packageManager.getActivityInfo(componentName, 0)
val orientation = info.screenOrientation  // Manifest에 설정한 값이 나옴

// 상수 이름표로 쓰는 경우 
setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_PORTRAIT)
```

### 3. setRequestOrientation() : 런타임 동적 변경
- 실제로 화면을 바꾸는 메서드. `ActivityInfo` 상수를 인자로 받는다.
```kotlin
// 유튜브처럼 전체화면 버튼 누르면 가로 전환
btnFullscreen.setOnClickListener {
    setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE)
}

// 전체화면 종료 시 세로 복귀
btnExitFullscreen.setOnClickListener {
    setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_PORTRAIT)
}

// Manifest 설정으로 리셋하고 싶을 때
setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_UNSPECIFIED)
```

## 4. sensor vs fullSensor vs fullUser

| | `sensor` | `fullSensor` | `fullUser` |
|---|---|---|---|
| 자동회전 OFF 시 | ❌ 회전 안 됨 | ✅ 강제 회전 | ❌ 회전 안 됨 |
| 자동회전 ON 시 | ✅ | ✅ | ✅ |
| 지원 방향 수 | 3방향 | **4방향** | **4방향** |
| 사용자 설정 존중 | ✅ | ❌ | ✅ |

> ⚠️ `FULL_SENSOR`는 사용자가 자동회전을 꺼놔도 강제로 돌아감.
> 영상, 게임 앱처럼 사용자 설정 무관하게 기울이면 따라가야 할 때 사용.

- sensor와 fullUser의 차이는 역방향이 지원되냐 안되냐의 차이

### 5. 번외
- `SCREEN_ORIENTATION_LOCKED` : 지금 설정된 방향으로 잠금
- `SCREEN_ORIENTATION_NOSENSOR` : 제조사가 정해놓은 기기 기본값으로 설정 (ex. 테블릿: 가로, 스마트폰 : 세로)
