# ✔️ Runnable
> Java가 제공하는 Thread 실행용 인터페이스

## 1️⃣ 사용방법 3가지
#### 1. 클래스로 구현
> 재사용이 필요할 때
```java
public class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("작업 실행: " + Thread.currentThread().getName());
    }
}

// 사용
Thread thread = new Thread(new MyTask());
thread.start();
```

#### 2. 익명 클래스 (Java 7 이하)
> 일회성 작업에서 클래스를 따로 만들기 싫을 때
```java
Thread thread = new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("익명 클래스로 실행");
    }
});
thread.start()
```

#### 3. 람다 클래스 (JAVA 8이상)
> Runnable은 함수형 인터페이스이기 때문에 람다로 간결하게 표현가능
```java
// 한 줄
Thread thread = new Thread(() -> System.out.println("람다로 실행"));
thread.start();

// 여러 줄
Runnable task = () -> {
    System.out.println("시작");
    // 작업 로직
    System.out.println("종료");
};
new Thread(task).start();
```

## 2️⃣ 주의할 점 : start vs run
```java
Runnable task = () -> {
    System.out.println("실행 스레드: " + Thread.currentThread().getName());
};

new Thread(task).run();   // 출력: "실행 스레드: main"

new Thread(task).start(); // 출력: "실행 스레드: Thread-0"
```
- run() 직접 호출: 일반 메서드 호출이다. 현재 스레드에서 동기적으로 실행되어 멀티스레드 효과가 전혀 없다.
- start() 호출: JVM이 새로운 OS 스레드를 생성하고 그 안에서 run()을 호출한다.

`public final boolean post (Runnable r)`

- Runnable r이 메시지 큐에 추가됩니다. 이 핸들러가 연결된 스레드에서 Runnable이 실행됩니다.
