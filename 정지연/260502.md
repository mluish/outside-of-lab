### Started Service
> 호출한 컴포넌트(Activity 등)의 생명주기와 완전히 독립되어 실행되는 서비스


### 1. 특징
- Android OS가 직접 생명주기를 관리한다.
- 호출자가 종료되어도 `stopSelf()` / `stopService()` 호출 전까지는 살아있다.

### 2. 생명주기
startService() 호출  -> onCreate() -> onStartCommand() -> 서비스 실행 중 -> onDestroy() -> 종료

- `onCreate()` : 서비스 인스턴스 최초 생성 시 1회만 호출. (초기화 작업 하는 곳)
- `onStartCommand()`: startService() 호출 마다 실행. 여러번 호출될 수 있음 (실제 작업 진입점)
- `onDestroy()`: stopSelf() / stopService 호출 시. 리소스 해제 위치

=> onCreate()는 서비스 인스턴스가 이미 존재하면 재호출되지 않는다. 반면, startService()를 여러 번 호출하면 onStartCommand()는 그 횟수만큼 호출됨

### 3. onStartCommand() 리턴값
시스템이 서비를 kill했다가 재시작할 때의 동작을 결정한다.

- `START_STICKY` : kill 후 재시작. Intent는 null로 전달
    - Intent 내용보다 서비스 지속이 중요할 때
    - ex) 음악 재생. 위치 추적
 
- `START_NOT_STICKY`: kill 하면 재시작하지 않음.
  - 이미 완료됐거나, 다음 startService() 서비스 호출 때 자연스럽게 재시작되는 작업

- `START_REDELIVEFR_INTENT`: kill 후 재시작 + 마지막 Intent 전달
  - 중단 없이 반드시 완료되어야하는 작업
  - ex) 파일 다운로드, 업로드

### 4. 스레드 주의 사항
Service는 기본적으로 메인 스레드에서 실행된다.
- 별도 프로세스나 스레드에서 실행되는 것이 아니다.
- onStartCommand()는 UI 스레드에서 호출한다.
- 네트워크 DB 작업을 직접 호출하면 ANR(5초 이상 블로킹)이 발생한다.

### 5. 코드 예시
- 앱이 꺼져도 다운로드는 계속 진행된다.

```koltin
// Activity
val intent = Intent(this, DownloadService::class.java)
    .putExtra("URL", "https://example.com/file.zip")
startService(intent)
// 여기서 홈 버튼 눌
```
```koltin
class DownloadService : Service() {
    private val scope = CoroutineScope(Dispatchers.IO + SupervisorJob())

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        val url = intent?.getStringExtra("URL") ?: return START_NOT_STICKY

        scope.launch {
            downloadFile(url)
            stopSelf(startId)  // 다운로드 끝나면 서비스 종료
        }
        return START_NOT_STICKY
    }

    override fun onDestroy() { scope.cancel(); super.onDestroy() }
    override fun onBind(intent: Intent?) = null
}
```
- 여러개의 파일을 다운로드 한다고 가정했을 때 stopSelf에 아이디를 부여하지 않으면 다음 파일도 작업이 중단됨
