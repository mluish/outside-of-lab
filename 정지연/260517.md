### 1. Frame 이란?
> 영상 한 장(이미지)
- 영상은 `Frame 1 -> Frame 2 -> Frame 3 -> ...` 를 빠르게 보여주는 것
  - ex) 30fps = 1초에 30장의 프레임 출력
  - ex) 60fps = 1초에 60장의 프레임 출력
 
❓ 왜 이미지라고 안하고 굳이 프레임이라고 하는지?
- 영상은 단순 이미지 모음이 아니라 시간에 따라 연속적으로 변하는 데이터이기 때문
- 어떤 이미지라기 보다는 언제 출력되냐를 강조하기에 image보다는 frame 용어를 사용

### 2. 스트리밍에서 프레임이 중요한 이유
> 영상 재생은 결국 프레임을 시간에 맞춰 출력하는 작업임
플레이어는 각 프레임을 **1. 네트워크로 받고, 2. 디코딩 하고, 3. 적절한 시간에 화면에 출력** 해야함

### 3. PTS(Presentation Time Stamp)
> "이 프레임을 언제 출력해야하는가?"를 나타낸 시간값
- 30fps라면 33ms마다 프레임이 출력된다.
| Frame | PTS  |
| ----- | ---- |
| A     | 0ms  |
| B     | 33ms |
| C     | 66ms |

### 4. 플레이어 내부 흐름
nework -> demux -> decoder(MediaCodec) -> render
압축된 영상 데이터 수신 -> 영상/오디오 분리 -> 압축 해제 -> PTS 시간 맞춰 출력

- 플레이어는 현재 재생시간(clock)과 PTS를 비교해서
    - 아직 이르면 기다리고
    - 늦었으면 프레임을 드롭하고
    - 맞으면 화면에 출력
 
❓ 영상과 오디오를 굳이 분리해야하는지?
- 영상과 오디오 압축 방식 자체가 완전히 다름
- 오디오는 소리 파형 압축
